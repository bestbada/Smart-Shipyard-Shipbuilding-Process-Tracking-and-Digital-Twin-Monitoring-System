# 스마트 조선소 공정 및 안전 모니터링 시스템 (디지털 트윈)

## 프로젝트 소개
2026 한이음 ICT 멘토링 (해운물류 트랙) 프로젝트임.

조선소 현장이 너무 넓어 관리자가 일일이 돌아다니며 확인하기 힘들다는 점에 착안함. 이동형 로봇(RC카)과 웹캠으로 현장 영상을 실시간 수집하고, 이를 AI 비전으로 분석하여 웹상의 3D 가상 조선소(디지털 트윈)에 띄워주는 시스템임. 관리자가 현장에 가지 않아도 원격으로 바로 상황을 파악하고 대처하는 것이 최종 목표임.

## 주요 기능
- 안전 관리: 작업자 안전모 미착용, 위험구역 접근, 쓰러짐(낙상) 실시간 감지
- 공정 확인: 선박 블록 조립 진행 상황 시각화
- 품질 및 화재 감시: 용접 불량(스패터)이나 초기 연기, 불꽃 탐지

## 상세 기술 스택 및 활용 방안

### 1. 현장 데이터 수집 (Edge)
- 고정형 웹캠(로지텍 C920): 삼각대로 높게 고정하여 공정 전체 상황을 위에서 내려다보는 조감(Bird-eye) 영상을 수집함. 웹캠으로 드론의 시야를 모사하여 대체함. (DJI Tello 드론은 선택 사항임)
- RC카(UGV): 사람이 가기 힘든 곳이나 특정 구역을 이동하며 사각지대 영상을 수집함.
- RTSP 프로토콜: 수집된 영상을 서버로 실시간 전송함. 실제 산업용 4족 보행 로봇이나 드론이 사용하는 통신 방식을 적용하여 향후 실제 장비 도입 시의 호환성을 고려함.

### 2. 영상 분석 (AI Vision)
- YOLOv8: 영상 속에서 안전모, 사람, 블록, 불꽃 등을 찾아내는 메인 모델임. 처리 속도가 빨라 실시간 관제에 적합하여 채택함.
- Pose Estimation: 작업자의 관절 포인트 17개를 추적함. 단순 착석과 사고로 인한 쓰러짐(낙상)을 정확히 구분하기 위해 사용함.
- FFmpeg: RTSP로 들어오는 실시간 영상 스트림을 AI가 분석할 수 있도록 프레임 단위로 변환함.

### 3. 서버 및 DB (Backend)
- Media Server (FFmpeg): RTSP로 들어오는 실시간 영상 스트림을 AI가 분석할 수 있도록 프레임 단위로 디코딩함.
- FastAPI: AI 분석 결과와 영상 데이터를 사용자에게 전달하는 메인 서버임. 파이썬 기반이라 AI 연동이 쉽고, 대량의 데이터 스트림을 비동기로 동시 처리하는 데 유리함.
- WebSocket: 서버와 웹 브라우저를 실시간으로 양방향 연결함. 사고 발생 시 즉각적인 경고를 띄우기 위해 도입함.
- MongoDB: 사고 발생 시간, 위치, 스냅샷 등 이벤트 로그를 저장함. 데이터 형태가 자유로운 NoSQL이라 로그 관리에 용이함.
- Docker: 완성된 시스템을 로컬이나 클라우드 환경 어디서든 쉽게 실행할 수 있도록 컨테이너 형태로 패키징함.

### 4. 관제 대시보드 (Frontend)
- Three.js: 웹 브라우저 안에 선박 블록 등 실제 조선소와 동일한 3D 가상 공간을 렌더링함. AI가 보낸 좌표를 바탕으로 3D 모델 위에 위험 요소를 시각화함.
- React.js & TypeScript: 대시보드의 전체적인 UI를 구성하고 데이터를 안정적으로 관리함.
- WebRTC: 경고 팝업 클릭 시 서버 딜레이 없이 현장 영상을 즉시 확인할 수 있도록 적용함.
